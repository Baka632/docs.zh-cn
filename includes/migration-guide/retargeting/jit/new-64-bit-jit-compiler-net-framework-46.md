### <a name="new-64-bit-jit-compiler-in-the-net-framework-46"></a><span data-ttu-id="b4c52-101">.NET Framework 4.6 中新的 64 位 JIT 编译器</span><span class="sxs-lookup"><span data-stu-id="b4c52-101">New 64-bit JIT compiler in the .NET Framework 4.6</span></span>

#### <a name="details"></a><span data-ttu-id="b4c52-102">详细信息</span><span class="sxs-lookup"><span data-stu-id="b4c52-102">Details</span></span>

<span data-ttu-id="b4c52-103">从 .NET Framework 4.6 开始，新的 64 位 JIT 编译器用于实时编译。</span><span class="sxs-lookup"><span data-stu-id="b4c52-103">Starting with the .NET Framework 4.6, a new 64-bit JIT compiler is used for just-in-time compilation.</span></span> <span data-ttu-id="b4c52-104">在某些情况下，比起使用 32 位编译器或较旧的 64 位 JIT 编译器运行应用，此举措会引发意外异常或发生其他行为。</span><span class="sxs-lookup"><span data-stu-id="b4c52-104">In some cases, an unexpected exception is thrown or a different behavior is observed than if an app is run using the 32-bit compiler or the older 64-bit JIT compiler.</span></span> <span data-ttu-id="b4c52-105">此更改不影响 32 位 JIT 编译器。</span><span class="sxs-lookup"><span data-stu-id="b4c52-105">This change does not affect the 32-bit JIT compiler.</span></span> <span data-ttu-id="b4c52-106">已知差异如下：</span><span class="sxs-lookup"><span data-stu-id="b4c52-106">The known differences include the following:</span></span>

- <span data-ttu-id="b4c52-107">在某些情况下，在启用优化的发布版本中，取消装箱操作可能会引发 <xref:System.NullReferenceException>。</span><span class="sxs-lookup"><span data-stu-id="b4c52-107">Under certain conditions, an unboxing operation may throw a <xref:System.NullReferenceException> in Release builds with optimization turned on.</span></span>
- <span data-ttu-id="b4c52-108">在某些情况下，在大型方法主体中执行生产代码可能会引发 <xref:System.StackOverflowException>。</span><span class="sxs-lookup"><span data-stu-id="b4c52-108">In some cases, execution of production code in a large method body may throw a <xref:System.StackOverflowException>.</span></span>
- <span data-ttu-id="b4c52-109">在某些情况下，传递给方法的结构将视为引用类型，而不是发布版本中的值类型。</span><span class="sxs-lookup"><span data-stu-id="b4c52-109">Under certain conditions, structures passed to a method are treated as reference types rather than as value types in Release builds.</span></span> <span data-ttu-id="b4c52-110">此问题的表现形式之一是，集合中各个项的显示顺序出现异常。</span><span class="sxs-lookup"><span data-stu-id="b4c52-110">One of the manifestations of this issue is that the individual items in a collection appear in an unexpected order.</span></span>
- <span data-ttu-id="b4c52-111">在某些情况下，如果启用了优化，无法正确比较 <xref:System.UInt16> 值与其高位集。</span><span class="sxs-lookup"><span data-stu-id="b4c52-111">Under certain conditions, the comparison of <xref:System.UInt16> values with their high bit set is incorrect if optimization is enabled.</span></span>
- <span data-ttu-id="b4c52-112">在某些情况下，尤其是在初始化数组值时，通过 <xref:System.Reflection.Emit.OpCodes.Initblk?displayProperty=nameWithType> IL 指令执行的内存初始化可能会使用不正确的值初始化内存。</span><span class="sxs-lookup"><span data-stu-id="b4c52-112">Under certain conditions, particularly when initializing array values, memory initialization by the <xref:System.Reflection.Emit.OpCodes.Initblk?displayProperty=nameWithType> IL instruction may initialize memory with an incorrect value.</span></span> <span data-ttu-id="b4c52-113">这可能会导致未经处理的异常抛出或输出不正确。</span><span class="sxs-lookup"><span data-stu-id="b4c52-113">This can result either in an unhandled exception or incorrect output.</span></span>
- <span data-ttu-id="b4c52-114">在某些极少数情况下，如果启用了编译器优化，条件位测试可能会返回错误的 <xref:System.Boolean> 值或引发异常。</span><span class="sxs-lookup"><span data-stu-id="b4c52-114">Under certain rare conditions, a conditional bit test can return the incorrect <xref:System.Boolean> value or throw an exception if compiler optimizations are enabled.</span></span>
- <span data-ttu-id="b4c52-115">在某些情况下，如果 `if` 语句用于在进入 `try` 块之前和从 `try` 块中退出之前测试条件，且在 `catch` 或 `finally` 块中计算的条件相同，那么新版 64 位 JIT 编译器会在优化代码时从 `catch` 或 `finally` 块中删除 `if` 条件。</span><span class="sxs-lookup"><span data-stu-id="b4c52-115">Under certain conditions, if an `if` statement is used to test for a condition before entering  a `try` block and in the exit from the `try` block, and the same condition is evaluated in the `catch` or `finally` block, the new 64-bit JIT compiler removes the `if` condition from the `catch` or `finally` block when it optimizes code.</span></span> <span data-ttu-id="b4c52-116">因此，`catch` 或 `finally` 块中的 `if` 语句代码会无条件地执行。</span><span class="sxs-lookup"><span data-stu-id="b4c52-116">As a result, code inside the `if` statement in the `catch` or `finally` block is executed unconditionally.</span></span>

<span data-ttu-id="b4c52-118">**已知问题的缓解措施**</span><span class="sxs-lookup"><span data-stu-id="b4c52-118">**Mitigation of known issues**</span></span> <br/> <span data-ttu-id="b4c52-119">如果遇到上面列出的问题，可以通过执行下列任一操作来解决：</span><span class="sxs-lookup"><span data-stu-id="b4c52-119">If you encounter the issues listed above, you can address them by doing any of the following:</span></span>

<span data-ttu-id="b4c52-126">**其他问题的缓解措施**</span><span class="sxs-lookup"><span data-stu-id="b4c52-126">**Mitigation of other issues**</span></span> <br/> <span data-ttu-id="b4c52-127">如果遇到的是旧版和新版 64 位 JIT 编译器编译的代码的其他任何行为差异，或是使用新版 64 位 JIT 编译器编译的应用程序的调试和发布版本的其他任何行为差异，可以使用旧版 64 位 JIT 编译器编译应用程序，具体操作如下：</span><span class="sxs-lookup"><span data-stu-id="b4c52-127">If you encounter any other difference in behavior between code compiled with the older 64-bit compiler and the new 64-bit JIT compiler, or between the debug and release versions of your app that are both compiled with the new 64-bit JIT compiler, you can do the following to compile your app with the older 64-bit JIT compiler:</span></span>

- <span data-ttu-id="b4c52-128">对于每个应用程序，可将 [<](~/docs/framework/configure-apps/file-schema/runtime/uselegacyjit-element.md) 元素添加到应用程序配置文件中。</span><span class="sxs-lookup"><span data-stu-id="b4c52-128">On a per-application basis, you can add the [<](~/docs/framework/configure-apps/file-schema/runtime/uselegacyjit-element.md) element to your application's configuration file.</span></span> <span data-ttu-id="b4c52-129">下面的代码禁用新版 64 位 JIT 编译器，改用旧版 64 位 JIT 编译器进行编译。</span><span class="sxs-lookup"><span data-stu-id="b4c52-129">The following disables compilation with the new 64-bit JIT compiler and instead uses the legacy 64-bit JIT compiler.</span></span>

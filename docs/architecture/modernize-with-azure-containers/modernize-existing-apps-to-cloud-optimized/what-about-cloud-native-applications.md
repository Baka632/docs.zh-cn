## <a name="cloud-native-applications-details"></a><span data-ttu-id="30503-121">云本机应用程序详细信息</span><span class="sxs-lookup"><span data-stu-id="30503-121">Cloud-native applications details</span></span>

<span data-ttu-id="30503-122">对于大型和任务关键型应用程序而言，云本地是一种更高级或更成熟的状态。</span><span class="sxs-lookup"><span data-stu-id="30503-122">Cloud-Native is a more advanced or mature state for large and mission-critical applications.</span></span> <span data-ttu-id="30503-123">云本机应用程序通常需要从头开始创建的体系结构和设计，而不是现代化现有应用程序。</span><span class="sxs-lookup"><span data-stu-id="30503-123">Cloud-Native applications usually require architecture and design that are created from scratch instead of by modernizing existing applications.</span></span> <span data-ttu-id="30503-124">云本机应用程序和更简单的云优化 Web 应用之间的主要区别是，建议使用云本机方法中的微服务体系结构。</span><span class="sxs-lookup"><span data-stu-id="30503-124">The key difference between a Cloud-Native application and a simpler Cloud-Optimized web app is the recommendation to use microservices architectures in a cloud-native approach.</span></span> <span data-ttu-id="30503-125">云优化应用也可以是整体式 Web 应用或 N 层应用。</span><span class="sxs-lookup"><span data-stu-id="30503-125">Cloud-Optimized apps can also be monolithic web apps or N-Tier apps.</span></span>

## <a name="what-about-microservices"></a><span data-ttu-id="30503-137">微服务呢？</span><span class="sxs-lookup"><span data-stu-id="30503-137">What about microservices?</span></span>

<span data-ttu-id="30503-138">在考虑组织的云本机应用程序时，了解微服务及其工作原理非常重要。</span><span class="sxs-lookup"><span data-stu-id="30503-138">Understanding microservices and how they work is important when you are considering cloud-native applications for your organization.</span></span>

<span data-ttu-id="30503-139">微服务体系结构是一种高级方法，可以将该方法用于从头开始创建的应用程序，也可以在将现有应用程序发展为云本机应用程序时将该方法用于应用程序。</span><span class="sxs-lookup"><span data-stu-id="30503-139">The microservices architecture is an advanced approach that you can use for applications that are created from scratch or when you evolve existing applications toward cloud-native applications.</span></span> <span data-ttu-id="30503-140">首先，可以将一些微服务添加到现有应用程序，以了解新的微服务范例。</span><span class="sxs-lookup"><span data-stu-id="30503-140">You can start by adding a few microservices to existing applications to learn about the new microservices paradigms.</span></span> <span data-ttu-id="30503-141">但很显然，你需要进行构建并编写代码，特别是对于这种类型的体系结构方法。</span><span class="sxs-lookup"><span data-stu-id="30503-141">But clearly, you need to architect and code, especially for this type of architectural approach.</span></span>

<span data-ttu-id="30503-142">但是，对于任何新的或新式应用程序，微服务不是必需的。</span><span class="sxs-lookup"><span data-stu-id="30503-142">However, microservices are not mandatory for any new or modern application.</span></span> <span data-ttu-id="30503-143">微服务并不是“万能的”，也不是创建每个应用程序的唯一最佳方法。</span><span class="sxs-lookup"><span data-stu-id="30503-143">Microservices are not a "magic bullet," and they aren't the single, best way to create every application.</span></span> <span data-ttu-id="30503-144">使用微服务的方式和时间取决于需要构建的应用程序的类型。</span><span class="sxs-lookup"><span data-stu-id="30503-144">How and when you use microservices depends on the type of application that you need to build.</span></span>

<span data-ttu-id="30503-145">微服务体系结构正成为基于多个独立子系统的分布式和大型或复杂的任务关键型应用程序的首选方法，其形式是自治服务。</span><span class="sxs-lookup"><span data-stu-id="30503-145">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications that are based on multiple, independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="30503-146">在基于微服务的体系结构中，将应用程序生成为可独立开发、测试、版本控制、部署和缩放的一系列服务。</span><span class="sxs-lookup"><span data-stu-id="30503-146">In a microservices-based architecture, an application is built as a collection of services that can be independently developed, tested, versioned, deployed, and scaled.</span></span> <span data-ttu-id="30503-147">使用这种方法，每个微服务可以包含任何相关的自治数据库。</span><span class="sxs-lookup"><span data-stu-id="30503-147">This approach can include any related, autonomous database per microservice.</span></span>

<span data-ttu-id="30503-148">有关可使用 .NET 实现的微服务体系结构的详细信息，请参阅可下载的 PDF 电子书 [.NET 微服务：适用于容器化 .NET 应用程序的体系结构](https://aka.ms/microservicesebook)。</span><span class="sxs-lookup"><span data-stu-id="30503-148">For a detailed look at a microservices architecture that you can implement by using .NET, see the downloadable PDF e-book [.NET microservices: Architecture for containerized .NET applications](https://aka.ms/microservicesebook).</span></span> <span data-ttu-id="30503-149">可以[在线](../../microservices/index.md)获得本指南。</span><span class="sxs-lookup"><span data-stu-id="30503-149">The guide also is available [online](../../microservices/index.md).</span></span>

<span data-ttu-id="30503-150">但即使是在微服务提供功能强大的独立部署、强子系统边界和技术多样性的情况下，它们也会带来许多新挑战。</span><span class="sxs-lookup"><span data-stu-id="30503-150">But even in scenarios in which microservices offer powerful capabilities-independent deployment, strong subsystem boundaries, and technology diversity-they also raise many new challenges.</span></span> <span data-ttu-id="30503-151">这些挑战与分布式应用程序开发相关，如零碎和独立的数据模型；实现微服务之间的可复原通信；需要最终一致性；以及操作复杂性。</span><span class="sxs-lookup"><span data-stu-id="30503-151">The challenges are related to distributed application development, such as fragmented and independent data models; achieving resilient communication between microservices; the need for eventual consistency; and operational complexity.</span></span> <span data-ttu-id="30503-152">与传统整体式应用程序相比，微服务加深了复杂性。</span><span class="sxs-lookup"><span data-stu-id="30503-152">Microservices introduce a higher level of complexity compared to traditional monolithic applications.</span></span>

<span data-ttu-id="30503-153">由于微服务体系结构的复杂性，因此只有特定方案和特定应用程序类型适用于基于微服务的应用程序。</span><span class="sxs-lookup"><span data-stu-id="30503-153">Because of the complexity of a microservices architecture, only specific scenarios and certain application types are suitable for microservice-based applications.</span></span> <span data-ttu-id="30503-154">其中包括具有多个不断发展的子系统的大型复杂应用程序。</span><span class="sxs-lookup"><span data-stu-id="30503-154">These include large and complex applications that have multiple, evolving subsystems.</span></span> <span data-ttu-id="30503-155">在这些情况下，值得投资更复杂的软件体系结构，因为它将增强长期灵活性并提高应用程序维护效率。</span><span class="sxs-lookup"><span data-stu-id="30503-155">In these cases, it's worth investing in a more complex software architecture, for increased long-term agility and more efficient application maintenance.</span></span> <span data-ttu-id="30503-156">但对于不太复杂的方案，最好使用整体式应用程序方法或更简单的 N 层方法。</span><span class="sxs-lookup"><span data-stu-id="30503-156">But for less complex scenarios, it might be better to continue with a monolithic application approach or simpler N-Tier approaches.</span></span>

<span data-ttu-id="30503-157">最后要注意的是，即使冒着重复此概念的风险，也不应在应用程序中将微服务用作“全有或全无”。</span><span class="sxs-lookup"><span data-stu-id="30503-157">As a final note, even at the risk of being repetitive about this concept, you shouldn't look at using microservices in your applications as "all-in or nothing at all."</span></span> <span data-ttu-id="30503-158">可以通过基于微服务添加全新小型方案来扩展和改进现有整体式应用程序。</span><span class="sxs-lookup"><span data-stu-id="30503-158">You can extend and evolve existing monolithic applications by adding new, small scenarios based on microservices.</span></span> <span data-ttu-id="30503-159">无需从头开始即可使用微服务体系结构方法。</span><span class="sxs-lookup"><span data-stu-id="30503-159">You don't need to start from scratch to start working with a microservices architecture approach.</span></span> <span data-ttu-id="30503-160">事实上，我们建议通过添加新方案来改进现有整体式或 N 层应用程序。</span><span class="sxs-lookup"><span data-stu-id="30503-160">In fact, we recommend that you evolve from using an existing monolithic or N-Tier application by adding new scenarios.</span></span> <span data-ttu-id="30503-161">最终，可以将应用程序细分为自治组件或微服务。</span><span class="sxs-lookup"><span data-stu-id="30503-161">Eventually, you can break down the application into autonomous components or microservices.</span></span> <span data-ttu-id="30503-162">可以沿微服务的方向逐步开始改进整体式应用程序。</span><span class="sxs-lookup"><span data-stu-id="30503-162">You can start evolving your monolithic applications in a microservices direction, step by step.</span></span>

<span data-ttu-id="30503-163">在任何情况下，本指南的其余部分都集中于“没有基于微服务的应用”，因为本指南主要旨在实现通常具有整体式或 N 层体系结构的现有应用的现代化。</span><span class="sxs-lookup"><span data-stu-id="30503-163">In any case, the rest of this present guidance focuses most of all on "no microservices-based apps" because this guidance is mainly targeting the modernization of existing apps that usually have monolithic or N-Tier architectures.</span></span>

## <a name="cloud-optimized-application-principles-and-tenets"></a><span data-ttu-id="eabfa-112">云优化应用程序原则和准则</span><span class="sxs-lookup"><span data-stu-id="eabfa-112">Cloud-Optimized application principles and tenets</span></span>

<span data-ttu-id="eabfa-113">云中的改进主要侧重于满足以下两个目标：通过提高灵活性，降低成本并改善业务增长。</span><span class="sxs-lookup"><span data-stu-id="eabfa-113">Improvements in the cloud are mostly focused on meeting two goals: Reduce costs and improve business growth by improving agility.</span></span> <span data-ttu-id="eabfa-114">这些目标是通过在发布和交付应用程序时简化过程和减少摩擦来实现的。</span><span class="sxs-lookup"><span data-stu-id="eabfa-114">These goals are achieved by simplifying processes and reducing friction when you release and ship applications.</span></span>

<span data-ttu-id="eabfa-115">如果你可以从其他本地应用程序灵活地自主开发应用程序，然后在云中发布、部署、自动缩放、监视应用程序和排除应用程序故障，则你的应用程序是云优化的。</span><span class="sxs-lookup"><span data-stu-id="eabfa-115">Your application is Cloud-Optimized if you can-in an agile manner-develop your app autonomously from other on-premises apps, and then release, deploy, autoscale, monitor, and troubleshoot your app in the cloud.</span></span>

<span data-ttu-id="eabfa-116">该项为“灵活性”  。</span><span class="sxs-lookup"><span data-stu-id="eabfa-116">The key is *agility*.</span></span> <span data-ttu-id="eabfa-117">你不能提供灵活性，除非你将任何部署到生产问题和开发/测试环境问题降低到绝对最少。</span><span class="sxs-lookup"><span data-stu-id="eabfa-117">You can't ship with agility unless you reduce to an absolute minimum any deployment-to-production issues and dev/test environment issues.</span></span> <span data-ttu-id="eabfa-118">专门为此目的设计了容器（特别是作为事实标准的 Docker）和托管服务。</span><span class="sxs-lookup"><span data-stu-id="eabfa-118">Containers (specifically, Docker, as a de facto standard) and managed services were designed specifically for this purpose.</span></span>

<span data-ttu-id="eabfa-119">若要实现灵活性，还需要基于 CI/CD 管道（这些管道发布到云中可缩放的平台）的自动化 DevOps 过程。</span><span class="sxs-lookup"><span data-stu-id="eabfa-119">To achieve agility, you also need automated DevOps processes that are based on CI/CD pipelines that release to scalable platforms in the cloud.</span></span> <span data-ttu-id="eabfa-120">部署到可缩放且可复原云平台（如 Azure 应用服务或 Azure Kubernetes 服务）的 CI/CD 平台（如 Azure DevOps Services 或 Jenkins）是在云中实现灵活性的关键技术。</span><span class="sxs-lookup"><span data-stu-id="eabfa-120">CI/CD platforms (like Azure DevOps Services or Jenkins) that deploy to a scalable and resilient cloud platform (like Azure App Service or Azure Kubernetes Service) are key technologies for achieving agility in the cloud.</span></span>

- <span data-ttu-id="eabfa-123">**容器**。</span><span class="sxs-lookup"><span data-stu-id="eabfa-123">**Containers**.</span></span> <span data-ttu-id="eabfa-124">通过容器，可以将应用程序依赖项包含到应用程序本身。</span><span class="sxs-lookup"><span data-stu-id="eabfa-124">Containers give you the ability to include application dependencies with the application itself.</span></span> <span data-ttu-id="eabfa-125">容器化大大减少了部署到生产环境时或在暂存环境中测试时可能遇到的问题数。</span><span class="sxs-lookup"><span data-stu-id="eabfa-125">Containerization significantly reduces the number of issues you might encounter when you deploy to production environments or test in staging environments.</span></span> <span data-ttu-id="eabfa-126">最终，容器改善了应用程序交付的灵活性。</span><span class="sxs-lookup"><span data-stu-id="eabfa-126">Ultimately, containers improve the agility of application delivery.</span></span>

- <span data-ttu-id="eabfa-127">**可复原且可缩放的云**。</span><span class="sxs-lookup"><span data-stu-id="eabfa-127">**Resilient and scalable cloud**.</span></span> <span data-ttu-id="eabfa-128">云提供了一个受管、弹性、可缩放且可复原的平台。</span><span class="sxs-lookup"><span data-stu-id="eabfa-128">The cloud provides a platform that is managed, elastic, scalable, and resilient.</span></span> <span data-ttu-id="eabfa-129">这些特性奠定了在持续交付中获得成本改进和提供高度可用且可靠的应用程序的基础。</span><span class="sxs-lookup"><span data-stu-id="eabfa-129">These characteristics are fundamental to gain cost improvements and ship highly available and reliable applications in a continuous delivery.</span></span> <span data-ttu-id="eabfa-130">托管服务（如托管数据库、托管缓存即服务 (CaaS) 和托管存储）是缓解应用程序维护成本的基本组成。</span><span class="sxs-lookup"><span data-stu-id="eabfa-130">Managed services like managed databases, managed cache as a service (CaaS), and managed storage are fundamental pieces in alleviating the maintenance costs of your application.</span></span>

- <span data-ttu-id="eabfa-131">**监视**。</span><span class="sxs-lookup"><span data-stu-id="eabfa-131">**Monitoring**.</span></span> <span data-ttu-id="eabfa-132">如果不使用合适的方法来检测和诊断异常以及应用程序性能问题，你将无法拥有可靠的应用程序。</span><span class="sxs-lookup"><span data-stu-id="eabfa-132">You can't have a reliable application without having a good way to detect and diagnose exceptions and application performance issues.</span></span> <span data-ttu-id="eabfa-133">需要通过应用程序性能管理和即时分析获取可付诸实施的见解。</span><span class="sxs-lookup"><span data-stu-id="eabfa-133">You need to get actionable insights through application performance management and instant analytics.</span></span>

- <span data-ttu-id="eabfa-134">**DevOps 区域性和持续交付**。</span><span class="sxs-lookup"><span data-stu-id="eabfa-134">**DevOps culture and continuous delivery**.</span></span> <span data-ttu-id="eabfa-135">采用 DevOps 做法需要文化变革，即团队不再在独立的孤岛中工作。</span><span class="sxs-lookup"><span data-stu-id="eabfa-135">Adopting DevOps practices requires a cultural change in which teams no longer work in independent silos.</span></span> <span data-ttu-id="eabfa-136">仅当开发和 IT 运营团队之间的增长协作受容器和 CI/CD 工具支持时，才可以使用 CI/CD 管道。</span><span class="sxs-lookup"><span data-stu-id="eabfa-136">CI/CD pipelines are possible only when there is an increased collaboration between development and IT operations teams, supported by containers and CI/CD tools.</span></span>

<span data-ttu-id="eabfa-142">概括而言，云优化应用程序是构建和管理应用程序的一种方法，该方法利用云计算模型，同时使用容器、托管云基础结构、可复原应用程序技术，监视、持续交付和 DevOps 的组合，所有这些都不需要对现有应用程序进行重新构建和重新编码。</span><span class="sxs-lookup"><span data-stu-id="eabfa-142">To summarize, a Cloud-Optimized application is an approach to building and managing applications that takes advantage of the cloud computing model, while using a combination of containers, managed cloud infrastructure, resilient application techniques, monitoring, continuous delivery, and DevOps, all without the need to rearchitect and recode your existing applications.</span></span>

## <a name="benefits-of-a-cloud-optimized-application"></a><span data-ttu-id="eabfa-146">云优化应用程序的益处</span><span class="sxs-lookup"><span data-stu-id="eabfa-146">Benefits of a Cloud-Optimized application</span></span>

<span data-ttu-id="eabfa-147">可以通过将现有应用程序转换为云优化应用程序（无需重新构建或编码）来获得以下好处：</span><span class="sxs-lookup"><span data-stu-id="eabfa-147">You can get the following benefits by converting an existing application to a Cloud-Optimized application (without rearchitecting or coding):</span></span>

- <span data-ttu-id="eabfa-148">**降低成本，因为托管基础结构由云提供商处理**。</span><span class="sxs-lookup"><span data-stu-id="eabfa-148">**Lower costs, because the managed infrastructure is handled by the cloud provider**.</span></span> <span data-ttu-id="eabfa-149">利用云的现成弹性、自动缩放和高可用性，云优化应用程序可以获得云的益处。</span><span class="sxs-lookup"><span data-stu-id="eabfa-149">Cloud-Optimized applications get the benefits of the cloud by using the cloud's out-of-the-box elasticity, autoscale, and high availability.</span></span> <span data-ttu-id="eabfa-150">益处不仅与计算功能（VM 和容器）相关，还取决于云中的资源，例如 DBaaS、CaaS 和应用程序可能需要的任何基础结构。</span><span class="sxs-lookup"><span data-stu-id="eabfa-150">Benefits are related not only to the compute features (VMs and containers), but also depend on the resources in the cloud, like DBaaS, CaaS, and any infrastructure an application might needed.</span></span>

- <span data-ttu-id="eabfa-151">**可复原应用程序和基础结构**。</span><span class="sxs-lookup"><span data-stu-id="eabfa-151">**Resilient application and infrastructure**.</span></span> <span data-ttu-id="eabfa-152">迁移到云时，需要接受暂时性故障；故障将在云中发生。</span><span class="sxs-lookup"><span data-stu-id="eabfa-152">When you migrate to the cloud, you need to embrace transient failures; failures will occur in the cloud.</span></span> <span data-ttu-id="eabfa-153">此外，云基础结构和硬件是“可更换的”，这会增加暂时性停机的机会。</span><span class="sxs-lookup"><span data-stu-id="eabfa-153">Also, cloud infrastructure and hardware are "replaceable," which increases opportunities for transient downtime.</span></span> <span data-ttu-id="eabfa-154">同时，实现复原和自动恢复的内部云功能和某些应用程序开发技术可以更轻松地在云中发生意外故障时进行恢复。</span><span class="sxs-lookup"><span data-stu-id="eabfa-154">At the same time, inner cloud capabilities and certain application development techniques that implement resiliency and automate recovery make it much easier to recover from unexpected failures in the cloud.</span></span>

- <span data-ttu-id="eabfa-155">**深入了解应用程序性能**。</span><span class="sxs-lookup"><span data-stu-id="eabfa-155">**Deeper insights into application performance**.</span></span> <span data-ttu-id="eabfa-156">Azure Application Insights 等云监视工具提供运行状况管理、日志记录和通知的可视化。</span><span class="sxs-lookup"><span data-stu-id="eabfa-156">Cloud monitoring tools like Azure Application Insights provide visualization for health management, logging, and notifications.</span></span> <span data-ttu-id="eabfa-157">审核日志使应用程序易于调试和审核，这是可靠的云应用程序的基础。</span><span class="sxs-lookup"><span data-stu-id="eabfa-157">Audit logs make applications easy to debug and audit, fundamental for a reliable cloud application.</span></span>

- <span data-ttu-id="eabfa-158">**应用程序可移植性，具有灵活部署**。</span><span class="sxs-lookup"><span data-stu-id="eabfa-158">**Application portability, with agile deployments**.</span></span> <span data-ttu-id="eabfa-159">容器（基于 Docker 引擎的 Linux 或 Windows 容器）提供最佳解决方案来避免云锁定的应用程序。</span><span class="sxs-lookup"><span data-stu-id="eabfa-159">Containers (either Linux or Windows containers based on Docker Engine) offer the best solution to avoiding a cloud-locked application.</span></span> <span data-ttu-id="eabfa-160">通过使用容器、Docker 主机和多云业务流程协调程序，你可以轻松地从一个环境或云移到另一个环境或云。</span><span class="sxs-lookup"><span data-stu-id="eabfa-160">By using containers, Docker hosts, and multi-cloud orchestrators, you can easily move from one environment or cloud to another.</span></span> <span data-ttu-id="eabfa-161">容器消除了通常出现在任何环境（暂存/测试/生产）的部署中的摩擦。</span><span class="sxs-lookup"><span data-stu-id="eabfa-161">Containers eliminate the friction that typically occurs in deployments to any environment (stage/test/production).</span></span>

<span data-ttu-id="eabfa-162">所有这些益处最终极大降低了端到端应用程序生命周期的成本。</span><span class="sxs-lookup"><span data-stu-id="eabfa-162">All of these benefits ultimately provide key cost reductions for your end-to-end application lifecycle.</span></span>

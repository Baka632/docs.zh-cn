 <span data-ttu-id="194f8-104">当派生类构造函数不调用基类构造函数时，Visual Basic 会尝试生成对无参数基类构造函数的隐式调用。</span><span class="sxs-lookup"><span data-stu-id="194f8-104">When a derived class constructor does not call a base class constructor, Visual Basic attempts to generate an implicit call to a parameterless base class constructor.</span></span> <span data-ttu-id="194f8-105">如果基类中没有无需参数即可调用的可访问的构造函数，则 Visual Basic 无法生成隐式调用。</span><span class="sxs-lookup"><span data-stu-id="194f8-105">If there is no accessible constructor in the base class that can be called without arguments, Visual Basic cannot generate an implicit call.</span></span> <span data-ttu-id="194f8-106">在这种情况下，所需的构造函数标记有 <xref:System.ObsoleteAttribute> 特性，因此 Visual Basic 不能调用它。</span><span class="sxs-lookup"><span data-stu-id="194f8-106">In this case, the required constructor is marked with the <xref:System.ObsoleteAttribute> attribute, so Visual Basic cannot call it.</span></span>

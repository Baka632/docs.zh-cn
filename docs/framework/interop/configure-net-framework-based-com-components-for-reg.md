## <a name="create-an-application-manifest"></a><span data-ttu-id="98764-110">创建应用程序清单</span><span class="sxs-lookup"><span data-stu-id="98764-110">Create an application manifest</span></span>  
  
1. <span data-ttu-id="98764-111">使用 XML 编辑器创建（或修改）COM 应用程序拥有的应用程序清单，该应用程序与一个或多个托管组件进行交互操作。</span><span class="sxs-lookup"><span data-stu-id="98764-111">Using an XML editor, create (or modify) the application manifest owned by the COM application that is interoperating with one or more managed components.</span></span>  
  
2. <span data-ttu-id="98764-112">在文件开头插入以下标准标头：</span><span class="sxs-lookup"><span data-stu-id="98764-112">Insert the following standard header at the beginning of the file:</span></span>  
  
    ```xml  
    <?xml version="1.0" encoding="UTF-8" standalone="yes"?>  
    <assembly xmlns="urn:schemas-microsoft-com:asm.v1" manifestVersion="1.0">
    </assembly>
    ```  
  
     <span data-ttu-id="98764-113">有关清单元素及其属性的信息，请参阅[应用程序清单](/windows/desktop/SbsCs/application-manifests)。</span><span class="sxs-lookup"><span data-stu-id="98764-113">For information about manifest elements and their attributes, see [Application Manifests](/windows/desktop/SbsCs/application-manifests).</span></span>  
  
3. <span data-ttu-id="98764-114">标识清单的所有者。</span><span class="sxs-lookup"><span data-stu-id="98764-114">Identify the owner of the manifest.</span></span> <span data-ttu-id="98764-115">以下示例中，`myComApp` 版本 1 拥有清单文件。</span><span class="sxs-lookup"><span data-stu-id="98764-115">In the following example, `myComApp` version 1 owns the manifest file.</span></span>  
  
    ```xml  
    <?xml version="1.0" encoding="UTF-8" standalone="yes"?>  
    <assembly xmlns="urn:schemas-microsoft-com:asm.v1" manifestVersion="1.0">  
      <assemblyIdentity type="win32"
                        name="myOrganization.myDivision.myComApp"
                        version="1.0.0.0"
                        processorArchitecture="msil"
      />
    </assembly>  
    ```  
  
4. <span data-ttu-id="98764-116">标识依赖程序集。</span><span class="sxs-lookup"><span data-stu-id="98764-116">Identify dependent assemblies.</span></span> <span data-ttu-id="98764-117">以下示例中，`myComApp` 依赖于 `myManagedComp`。</span><span class="sxs-lookup"><span data-stu-id="98764-117">In the following example, `myComApp` depends on `myManagedComp`.</span></span>  
  
    ```xml  
    <?xml version="1.0" encoding="UTF-8" standalone="yes"?>  
    <assembly xmlns="urn:schemas-microsoft-com:asm.v1" manifestVersion="1.0">  
      <assemblyIdentity type="win32"
                        name="myOrganization.myDivision.myComApp"
                        version="1.0.0.0"
                        processorArchitecture="x86"
                        publicKeyToken="8275b28176rcbbef"  
      />  
      <dependency>  
        <dependentAssembly>  
          <assemblyIdentity type="win32"
                        name="myOrganization.myDivision.myManagedComp"
                        version="6.0.0.0"
                        processorArchitecture="X86"
                        publicKeyToken="8275b28176rcbbef"  
          />  
        </dependentAssembly>  
      </dependency>  
    </assembly>  
    ```  
  
5. <span data-ttu-id="98764-118">保存并命名清单文件。</span><span class="sxs-lookup"><span data-stu-id="98764-118">Save and name the manifest file.</span></span> <span data-ttu-id="98764-119">应用程序清单的名称是程序集可执行文件的名称后加 .manifest 扩展名。</span><span class="sxs-lookup"><span data-stu-id="98764-119">The name of an application manifest is the name of the assembly executable followed by the .manifest extension.</span></span> <span data-ttu-id="98764-120">例如，myComApp.exe 的应用程序清单文件名称是 myComApp.exe.manifest。</span><span class="sxs-lookup"><span data-stu-id="98764-120">For example, the application manifest file name for myComApp.exe is myComApp.exe.manifest.</span></span>  
  
<span data-ttu-id="98764-121">可在与 COM 应用程序相同的目录中安装应用程序清单。</span><span class="sxs-lookup"><span data-stu-id="98764-121">You can install an application manifest in the same directory as the COM application.</span></span> <span data-ttu-id="98764-122">或者，可将其作为资源添加到应用程序的 .exe 文件。</span><span class="sxs-lookup"><span data-stu-id="98764-122">Alternatively, you can add it as a resource to the application's .exe file.</span></span> <span data-ttu-id="98764-123">有关详细信息，请参阅[关于并行程序集](/windows/desktop/SbsCs/about-side-by-side-assemblies-)。</span><span class="sxs-lookup"><span data-stu-id="98764-123">For more information, see [About Side-by-Side Assemblies](/windows/desktop/SbsCs/about-side-by-side-assemblies-).</span></span>

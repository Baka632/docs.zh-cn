## <a name="state-management-and-process-recycling"></a><span data-ttu-id="3e211-115">状态管理和进程回收</span><span class="sxs-lookup"><span data-stu-id="3e211-115">State Management and Process Recycling</span></span>

> [!NOTE]
> <span data-ttu-id="3e211-119">WCF 用于消息层可靠性和安全性的协议利用可变的内存中状态。</span><span class="sxs-lookup"><span data-stu-id="3e211-119">The protocols WCF uses for message-layer reliability and security make use of the volatile in-memory state.</span></span> <span data-ttu-id="3e211-120">WCF 可靠会话和安全会话可能由于应用程序回收而意外终止。</span><span class="sxs-lookup"><span data-stu-id="3e211-120">WCF reliable sessions and security sessions may terminate unexpectedly due to application recycles.</span></span> <span data-ttu-id="3e211-121">使用这些协议的 IIS 托管的应用程序应该依赖于 WCF 提供的会话密钥来关联应用层状态 (例如，应用层构造或自定义相关标头) 或为托管应用程序禁用 IIS 进程回收。</span><span class="sxs-lookup"><span data-stu-id="3e211-121">IIS-hosted applications that make use of these protocols should either depend on something other than the WCF-provided session key for correlating application-layer state (for example, an application-layer construct or custom correlation header) or disable IIS process recycling for the hosted application.</span></span>  
  
## <a name="optimizing-performance-in-middle-tier-scenarios"></a><span data-ttu-id="3e211-122">在中间层方案中优化性能</span><span class="sxs-lookup"><span data-stu-id="3e211-122">Optimizing Performance in Middle-Tier Scenarios</span></span>  
 <span data-ttu-id="3e211-123">若要在 *中间层方案*中实现最佳性能（一项服务，该服务为响应传入消息而调用其他服务），请一次将 WCF 服务客户端实例化到远程服务，并跨多个传入请求重复使用它。</span><span class="sxs-lookup"><span data-stu-id="3e211-123">For optimal performance in a *middle-tier scenario*—a service that calls out to other services in response to incoming messages—instantiate the WCF service client to the remote service once and reuse it across multiple incoming requests.</span></span> <span data-ttu-id="3e211-124">实例化 WCF 服务客户端是一项成本高昂的操作，相对于对预先存在的客户端实例进行服务调用，中间层方案通过跨请求缓存远程客户端来产生明显的性能提升。</span><span class="sxs-lookup"><span data-stu-id="3e211-124">Instantiating WCF service clients is an expensive operation relative to making a service call on a pre-existing client instance, and middle-tier scenarios produce distinct performance gains by caching remote clients across requests.</span></span> <span data-ttu-id="3e211-125">WCF 服务客户端是线程安全的，因此不必跨多个线程同步对客户端的访问。</span><span class="sxs-lookup"><span data-stu-id="3e211-125">WCF service clients are thread-safe, so it is not necessary to synchronize access to a client across multiple threads.</span></span>

 <span data-ttu-id="3e211-131">若要确保服务元数据中出现正确的主机名，WCF 将生成，请将承载 WCF 服务的 IIS 网站的默认标识配置为使用显式主机名。</span><span class="sxs-lookup"><span data-stu-id="3e211-131">To ensure that the correct hostname appears in the service metadata WCF generates, configure the default identity for the IIS Web site that hosts WCF services to use an explicit hostname.</span></span> <span data-ttu-id="3e211-132">例如，驻留在场中的计算机 `www.contoso.com` 应使用 \*：80： www .com FOR HTTP 的 IIS 网站绑定，并使用 \* ：443： www .COM for HTTPS。</span><span class="sxs-lookup"><span data-stu-id="3e211-132">For example, computers that reside inside of the `www.contoso.com` farm should use an IIS site binding of \*:80:www.contoso.com for HTTP and \*:443:www.contoso.com for HTTPS.</span></span>

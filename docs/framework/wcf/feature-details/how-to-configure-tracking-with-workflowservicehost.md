### <a name="configure-tracking-in-configuration"></a><span data-ttu-id="89d7a-105">在配置中配置跟踪</span><span class="sxs-lookup"><span data-stu-id="89d7a-105">Configure Tracking in Configuration</span></span>  
  
1. <span data-ttu-id="89d7a-106"><xref:System.Activities.Tracking.EtwTrackingParticipant> `behavior` 在配置文件中使用 <> 元素添加，如下面的示例中所示。</span><span class="sxs-lookup"><span data-stu-id="89d7a-106">Add the <xref:System.Activities.Tracking.EtwTrackingParticipant> using the <`behavior`> element in a configuration file, as shown in the following example.</span></span>

    ```xml  
    <behaviors>  
       <serviceBehaviors>  
         <behavior>  
           <etwTracking profileName="Sample Tracking Profile" />  
         </behavior>
       </serviceBehaviors>  
    </behaviors>  
    ```  
  
    > [!NOTE]
    > <span data-ttu-id="89d7a-107">上面的配置示例使用的是简化配置。</span><span class="sxs-lookup"><span data-stu-id="89d7a-107">The preceding configuration sample is using simplified configuration.</span></span> <span data-ttu-id="89d7a-108">有关详细信息，请参阅 [简化配置](../simplified-configuration.md)。</span><span class="sxs-lookup"><span data-stu-id="89d7a-108">For more information, see [Simplified Configuration](../simplified-configuration.md).</span></span>  
  
     <span data-ttu-id="89d7a-109">上面的配置示例添加一个 <xref:System.Activities.Tracking.EtwTrackingParticipant>，并指定一个跟踪配置文件名称。</span><span class="sxs-lookup"><span data-stu-id="89d7a-109">The preceding configuration sample adds a <xref:System.Activities.Tracking.EtwTrackingParticipant> and specifies a tracking profile name.</span></span> <span data-ttu-id="89d7a-110">跟踪配置文件是在 `trackingProfile` <> 元素内的 <> 元素中创建的 `tracking` 。</span><span class="sxs-lookup"><span data-stu-id="89d7a-110">Tracking profiles are created in a <`trackingProfile`> element within a <`tracking`> element.</span></span> <span data-ttu-id="89d7a-111">跟踪配置文件包含跟踪查询，这些查询允许跟踪参与者订阅工作流实例的状态在运行时发生更改时发出的工作流事件。</span><span class="sxs-lookup"><span data-stu-id="89d7a-111">The tracking profile contains tracking queries that permit a tracking participant to subscribe to workflow events that are emitted when the state of a workflow instance changes at runtime.</span></span> <span data-ttu-id="89d7a-112">下面的示例演示如何创建跟踪配置文件。</span><span class="sxs-lookup"><span data-stu-id="89d7a-112">The following example shows how to create a tracking profile.</span></span>  
  
    ```xml

        <tracking>
         <trackingProfile name="Sample Tracking Profile">  
            <workflow activityDefinitionId="*">  
               <workflowInstanceQueries>  
                 <workflowInstanceQuery>  
                    <states>  
                       <state name="Started"/>  
                       <state name="Completed"/>  
                    </states>  
                </workflowInstanceQuery>  
             </workflowInstanceQueries>  
           </workflow>  
         </trackingProfile>
       </tracking>  
    </system.serviceModel>  
    ```

# <a name="weakly-typed-json-serialization-sample"></a><span data-ttu-id="1b1cc-102">弱类型 JSON 序列化示例</span><span class="sxs-lookup"><span data-stu-id="1b1cc-102">Weakly-typed JSON Serialization Sample</span></span>

<span data-ttu-id="1b1cc-103">将用户定义的类型序列化为给定的连网格式，或者将连网格式反序列为原来的用户定义的类型时，给定的用户定义的类型必须在服务和客户端上可用。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-103">When serializing a user-defined type to a given wire format, or deserializing a wire format back into a user-defined type, the given user-defined type must be available on both the service and the client.</span></span> <span data-ttu-id="1b1cc-104">通常，为实现此目的，系统将 <xref:System.Runtime.Serialization.DataContractAttribute> 属性应用于这些用户定义的类型，并将 <xref:System.Runtime.Serialization.DataMemberAttribute> 属性应用于这些类型的成员。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-104">Usually to accomplish this, the <xref:System.Runtime.Serialization.DataContractAttribute> attribute is applied to these user-defined types and the <xref:System.Runtime.Serialization.DataMemberAttribute> attribute is applied to their members.</span></span> <span data-ttu-id="1b1cc-105">处理 JavaScript 对象符号 (JSON) 对象时，该机制同样适用，如主题 [How to: Serialize and Deserialize JSON Data](../feature-details/how-to-serialize-and-deserialize-json-data.md)中所述。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-105">This mechanism also applies when working with JavaScript Object Notation (JSON) objects, as described in the topic [How to: Serialize and Deserialize JSON Data](../feature-details/how-to-serialize-and-deserialize-json-data.md).</span></span>  
  
 <span data-ttu-id="1b1cc-106">在某些情况下，Windows Communication Foundation (WCF) 服务或客户端必须访问由不受开发人员控制的服务或客户端生成的 JSON 对象。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-106">In some scenarios, a Windows Communication Foundation (WCF) service or client must access JSON objects generated by a service or client that is outside of the control of the developer.</span></span> <span data-ttu-id="1b1cc-107">随着越来越多的 Web 服务公开 JSON Api，WCF 开发人员可能不需要构造任意 JSON 对象要反序列化的本地用户定义类型。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-107">As more Web services publicly expose JSON APIs, it can become impractical for the WCF developer to construct local user-defined types into which to deserialize arbitrary JSON objects.</span></span> <span data-ttu-id="1b1cc-108">本示例提供了一种机制，使 WCF 开发人员无需创建用户定义的类型即可使用反序列化的任意 JSON 对象。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-108">This sample provides a mechanism that enables WCF developers to work with deserialized, arbitrary JSON objects, without creating user-defined types.</span></span> <span data-ttu-id="1b1cc-109">这称为 JSON 对象的“弱类型序列化”  ，因为 JSON 对象反序列化为的类型在编译时是未知的。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-109">This is known as *weakly-typed serialization* of JSON objects, because the type into which a JSON object deserializes is not known at compile time.</span></span>

> <span data-ttu-id="1b1cc-110">本主题的最后介绍了此示例的设置过程和生成说明。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-110">The setup procedure and build instructions for this sample are located at the end of this topic.</span></span>  
  
 <span data-ttu-id="1b1cc-111">例如，一个公共 Web 服务 API 返回以下 JSON 对象，该对象描述有关该服务的用户的一些信息。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-111">For example, a public Web service API returns the following JSON object, which describes some information about a user of the service.</span></span>  
  
```json  
{"personal": {"name": "Paul", "age": 23, "height": 1.7, "isSingle": true, "luckyNumbers": [5,17,21]}, "favoriteBands": ["Band ABC", "Band XYZ"]}  
```  
  
 <span data-ttu-id="1b1cc-112">若要反序列化此对象，WCF 客户端必须实现以下用户定义的类型。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-112">To deserialize this object, a WCF client must implement the following user-defined types.</span></span>  
  
```csharp  
[DataContract]  
 public class MemberProfile  
 {  
     [DataMember]  
     public PersonalInfo personal;  
  
     [DataMember]  
     public string[] favoriteBands;  
 }  
  
 [DataContract]  
 public class PersonalInfo  
 {  
     [DataMember]  
     public string name;  
  
     [DataMember]  
     public int age;  
  
     [DataMember]  
     public double height;  
  
     [DataMember]  
     public bool isSingle;  
  
     [DataMember]  
     public int[] luckyNumbers;  
 }  
```  
  
 <span data-ttu-id="1b1cc-113">这可能会很麻烦，尤其是客户端必须处理多种 JSON 对象时。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-113">This can be cumbersome, especially if the client has to handle more than one type of JSON object.</span></span>  
  
 <span data-ttu-id="1b1cc-114">此示例提供的 `JsonObject` 类型引入了以弱类型表示的反序列化的 JSON 对象。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-114">The `JsonObject` type provided by this sample introduces a weakly-typed representation of the deserialized JSON object.</span></span> <span data-ttu-id="1b1cc-115">`JsonObject` 依赖于 JSON 对象与 .NET Framework 字典之间的自然映射和 JSON 数组与 .NET Framework 数组之间的映射。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-115">`JsonObject` relies on the natural mapping between JSON objects and .NET Framework dictionaries, and the mapping between JSON arrays and .NET Framework arrays.</span></span> <span data-ttu-id="1b1cc-116">下面的代码演示 `JsonObject` 类型。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-116">The following code shows the `JsonObject` type.</span></span>  
  
```csharp  
// Instantiation of JsonObject json omitted  
  
string name = json["root"]["personal"]["name"];  
int age = json["root"]["personal"]["age"];  
double height = json["root"]["personal"]["height"];  
bool isSingle = json["root"]["personal"]["isSingle"];  
int[] luckyNumbers = {  
                                     json["root"]["personal"]["luckyNumbers"][0],  
                                     json["root"]["personal"]["luckyNumbers"][1],  
                                     json["root"]["personal"]["luckyNumbers"][2]
                                 };  
string[] favoriteBands = {  
                                        json["root"]["favoriteBands"][0],  
                                        json["root"]["favoriteBands"][1]  
                                    };  
```  
  
 <span data-ttu-id="1b1cc-117">请注意，您无需在编译时声明 JSON 对象和数组的类型便可以“浏览”它们。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-117">Note that you can "browse" JSON objects and arrays without the need to declare their type at compile time.</span></span> <span data-ttu-id="1b1cc-118">有关顶级 `["root"]` 对象的要求的说明，请参阅主题 [Mapping Between JSON and XML](../feature-details/mapping-between-json-and-xml.md)中所述。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-118">For an explanation of the requirement for the top-level `["root"]` object, see the topic [Mapping Between JSON and XML](../feature-details/mapping-between-json-and-xml.md).</span></span>  
  
> [!NOTE]
> <span data-ttu-id="1b1cc-119">`JsonObject` 类仅作为示例提供，</span><span class="sxs-lookup"><span data-stu-id="1b1cc-119">The `JsonObject` class is provided as an example only.</span></span> <span data-ttu-id="1b1cc-120">它未经全面测试，不应在生产环境中使用。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-120">It has not been thoroughly tested, and should not be used in production environments.</span></span> <span data-ttu-id="1b1cc-121">弱类型 JSON 序列化的一个明显含义是在处理 `JsonObject`时缺乏类型安全。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-121">An obvious implication of weakly-typed JSON serialization is the lack of type-safety when working with `JsonObject`.</span></span>  
  
 <span data-ttu-id="1b1cc-122">若要使用 `JsonObject` 类型，客户端操作协定必须使用 <xref:System.ServiceModel.Channels.Message> 作为其返回类型。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-122">To use the `JsonObject` type, the client operation contract must use <xref:System.ServiceModel.Channels.Message> as its return type.</span></span>  
  
```csharp  
[ServiceContract]  
    interface IClientSideProfileService  
    {  
        // There is no need to write a DataContract for the complex type returned by the service.  
        // The client will use a JsonObject to browse the JSON in the received message.  
  
        [OperationContract]  
        [WebGet(ResponseFormat = WebMessageFormat.Json)]  
        Message GetMemberProfile();  
    }  
```  
  
 <span data-ttu-id="1b1cc-123">随后实例化 `JsonObject` ，如下面的代码所示。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-123">The `JsonObject` is then instantiated as shown in the following code.</span></span>  
  
```csharp  
// Code to instantiate IClientSideProfileService channel omitted…  
  
// Make a request to the service and obtain the Json response  
XmlDictionaryReader reader = channel.GetMemberProfile().GetReaderAtBodyContents();  
  
// Go through the Json as though it is a dictionary. There is no need to map it to a .NET CLR type.  
JsonObject json = new JsonObject(reader);  
```  
  
 <span data-ttu-id="1b1cc-124">`JsonObject` 构造函数采用通过 <xref:System.Xml.XmlDictionaryReader>方法获取的 <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> 。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-124">The `JsonObject` constructor takes a <xref:System.Xml.XmlDictionaryReader>, which is obtained through the <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> method.</span></span> <span data-ttu-id="1b1cc-125">该读取器包含客户端接收的 JSON 消息的 XML 表示形式。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-125">The reader contains an XML representation of the JSON message received by the client.</span></span> <span data-ttu-id="1b1cc-126">有关详细信息，请参阅 [JSON 和 XML 之间的映射](../feature-details/mapping-between-json-and-xml.md)主题。</span><span class="sxs-lookup"><span data-stu-id="1b1cc-126">For more information, see the topic [Mapping Between JSON and XML](../feature-details/mapping-between-json-and-xml.md).</span></span>

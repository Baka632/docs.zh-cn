 <span data-ttu-id="7fd12-117">异步线程中止出现问题，因为它们在目标线程的执行过程中是在不可预测的点被处理的。</span><span class="sxs-lookup"><span data-stu-id="7fd12-117">Asynchronous thread aborts present a problem because they are processed at unpredictable points in the target thread's execution.</span></span> <span data-ttu-id="7fd12-118">若要避免此问题，所编写的代码（为了在可能以此方式被中止的线程上运行）需要在几乎每个代码行上处理 <xref:System.Threading.ThreadAbortException>，负责将应用程序数据放回到一致状态。</span><span class="sxs-lookup"><span data-stu-id="7fd12-118">To avoid this, code written to run on a thread that might be aborted in this manner would need to handle a <xref:System.Threading.ThreadAbortException> at almost every line of code, taking care to put application data back into a consistent state.</span></span> <span data-ttu-id="7fd12-119">在编写代码时考虑到此问题，或编写防止可能出现的所有情况的代码都是不现实的。</span><span class="sxs-lookup"><span data-stu-id="7fd12-119">It is not realistic to expect code to be written with this problem in mind or to write code that protects against all possible circumstances.</span></span>

## <a name="example"></a><span data-ttu-id="7fd12-135">示例</span><span class="sxs-lookup"><span data-stu-id="7fd12-135">Example</span></span>

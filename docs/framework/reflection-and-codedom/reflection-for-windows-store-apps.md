<span data-ttu-id="86cfc-105">从 .NET Framework 4.5 开始，.NET Framework 包含一系列用于 Windows 8.x 应用商店应用的反射类型和成员。</span><span class="sxs-lookup"><span data-stu-id="86cfc-105">Starting with the .NET Framework 4.5, the .NET Framework includes a set of reflection types and members for use in Windows 8.x Store apps.</span></span> <span data-ttu-id="86cfc-106">这些类型和成员可从完整的 .NET Framework 以及适用于 Windows 应用商店应用的 .NET 中获取。</span><span class="sxs-lookup"><span data-stu-id="86cfc-106">These types and members are available in the full .NET Framework as well as in the .NET for Windows Store apps.</span></span> <span data-ttu-id="86cfc-107">本文档介绍 .NET Framework 4 及更早版本中这些类型和成员与其对应项之间的主要差异。</span><span class="sxs-lookup"><span data-stu-id="86cfc-107">This document explains the major differences between these and their counterparts in the .NET Framework 4 and earlier versions.</span></span>  
  
 <span data-ttu-id="86cfc-108">如果要创建 Windows 8.x 应用商店应用，则必须在适用于 Windows 8.x 应用商店应用的 .NET 中使用反射类型和成员。</span><span class="sxs-lookup"><span data-stu-id="86cfc-108">If you are creating a Windows 8.x Store app, you must use the reflection types and members in the .NET for Windows 8.x Store apps.</span></span> <span data-ttu-id="86cfc-109">这些类型和成员也可用于桌面应用，但不要求必须用于，因此你可以针对这两种类型的应用使用相同的代码。</span><span class="sxs-lookup"><span data-stu-id="86cfc-109">These types and members are also available, but not required, for use in desktop apps, so you can use the same code for both types of apps.</span></span>  
  
## <a name="typeinfo-and-assembly-loading"></a><span data-ttu-id="86cfc-110">TypeInfo 和程序集加载</span><span class="sxs-lookup"><span data-stu-id="86cfc-110">TypeInfo and Assembly Loading</span></span>  

 <span data-ttu-id="86cfc-111">在适用于 Windows 8.x 应用商店应用的 .NET 中，<xref:System.Reflection.TypeInfo> 类包含 .NET Framework 4 <xref:System.Type> 类的某些功能。</span><span class="sxs-lookup"><span data-stu-id="86cfc-111">In the .NET for Windows 8.x Store apps, the <xref:System.Reflection.TypeInfo> class contains some of the functionality of the .NET Framework 4 <xref:System.Type> class.</span></span> <span data-ttu-id="86cfc-112"><xref:System.Type> 对象表示对类型定义的引用，而 <xref:System.Reflection.TypeInfo> 对象表示该类型定义本身。</span><span class="sxs-lookup"><span data-stu-id="86cfc-112">A <xref:System.Type> object represents a reference to a type definition, whereas a <xref:System.Reflection.TypeInfo> object represents the type definition itself.</span></span> <span data-ttu-id="86cfc-113">这使你能够操纵 <xref:System.Type> 对象，而不一定需要运行时加载它们引用的程序集。</span><span class="sxs-lookup"><span data-stu-id="86cfc-113">This enables you to manipulate <xref:System.Type> objects without necessarily requiring the runtime to load the assembly they reference.</span></span> <span data-ttu-id="86cfc-114">获取关联的 <xref:System.Reflection.TypeInfo> 对象将强制加载程序集。</span><span class="sxs-lookup"><span data-stu-id="86cfc-114">Getting the associated <xref:System.Reflection.TypeInfo> object forces the assembly to load.</span></span>  
  
 <span data-ttu-id="86cfc-115"><xref:System.Reflection.TypeInfo> 包含许多在 <xref:System.Type> 上可用的成员，并且适用于 Windows 8.x 应用商店应用的 .NET 中的许多反射属性返回 <xref:System.Reflection.TypeInfo> 对象的集合。</span><span class="sxs-lookup"><span data-stu-id="86cfc-115"><xref:System.Reflection.TypeInfo> contains many of the members available on <xref:System.Type>, and many of the reflection properties in the .NET for Windows 8.x Store apps return collections of <xref:System.Reflection.TypeInfo> objects.</span></span> <span data-ttu-id="86cfc-116">若要从 <xref:System.Type> 对象获取 <xref:System.Reflection.TypeInfo> 对象，请使用 <xref:System.Reflection.IReflectableType.GetTypeInfo%2A> 方法。</span><span class="sxs-lookup"><span data-stu-id="86cfc-116">To get a <xref:System.Reflection.TypeInfo> object from a <xref:System.Type> object, use the <xref:System.Reflection.IReflectableType.GetTypeInfo%2A> method.</span></span>  
  
## <a name="query-methods"></a><span data-ttu-id="86cfc-117">查询方法</span><span class="sxs-lookup"><span data-stu-id="86cfc-117">Query Methods</span></span>  

 <span data-ttu-id="86cfc-118">在适用于 Windows 8.x 应用商店应用的 .NET 中，请使用返回 <xref:System.Collections.Generic.IEnumerable%601> 集合的反射属性，而不是返回数组的方法。</span><span class="sxs-lookup"><span data-stu-id="86cfc-118">In the .NET for Windows 8.x Store apps, you use the reflection properties that return <xref:System.Collections.Generic.IEnumerable%601> collections instead of methods that return arrays.</span></span> <span data-ttu-id="86cfc-119">反射上下文可以实现这些大型程序集或类型集合的延迟遍历。</span><span class="sxs-lookup"><span data-stu-id="86cfc-119">Reflection contexts can implement lazy traversal of these collections for large assemblies or types.</span></span>  
  
 <span data-ttu-id="86cfc-120">反射属性将仅返回特定对象上声明的方法，而不会遍历继承树。</span><span class="sxs-lookup"><span data-stu-id="86cfc-120">The reflection properties return only the declared methods on a particular object instead of traversing the inheritance tree.</span></span> <span data-ttu-id="86cfc-121">此外，它们不使用 <xref:System.Reflection.BindingFlags> 参数进行筛选。</span><span class="sxs-lookup"><span data-stu-id="86cfc-121">Moreover, they do not use <xref:System.Reflection.BindingFlags> parameters for filtering.</span></span> <span data-ttu-id="86cfc-122">相反，通过对返回的集合使用 LINQ 查询，可以在用户代码中进行筛选。</span><span class="sxs-lookup"><span data-stu-id="86cfc-122">Instead, filtering takes place in user code, by using LINQ queries on the returned collections.</span></span> <span data-ttu-id="86cfc-123">对于源于运行时的反射对象（例如，作为 `typeof(Object)` 的结果），最好通过使用 <xref:System.Reflection.RuntimeReflectionExtensions> 类的帮助程序方法实现继承树的遍历。</span><span class="sxs-lookup"><span data-stu-id="86cfc-123">For reflection objects that originate with the runtime (for example, as the result of `typeof(Object)`), traversing the inheritance tree is best accomplished by using the helper methods of the <xref:System.Reflection.RuntimeReflectionExtensions> class.</span></span> <span data-ttu-id="86cfc-124">自定义反射上下文中的对象的使用者不能使用这些方法，并且必须遍历继承树本身。</span><span class="sxs-lookup"><span data-stu-id="86cfc-124">Consumers of objects from customized reflection contexts cannot use these methods, and must traverse the inheritance tree themselves.</span></span>

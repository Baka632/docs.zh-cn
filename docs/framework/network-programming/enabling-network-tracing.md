# <a name="enabling-network-tracing"></a><span data-ttu-id="ba728-103">启用网络跟踪</span><span class="sxs-lookup"><span data-stu-id="ba728-103">Enabling Network Tracing</span></span>

<span data-ttu-id="ba728-104">网络跟踪允许访问有关方法调用的信息，以及有关托管应用程序所生成的网络流量的信息。</span><span class="sxs-lookup"><span data-stu-id="ba728-104">Network tracing provides access to information about method invocations and network traffic generated by a managed application.</span></span> <span data-ttu-id="ba728-105">必须完成以下任务才能在应用程序中启用网络跟踪：</span><span class="sxs-lookup"><span data-stu-id="ba728-105">You must complete the following tasks to enable network tracing in your application:</span></span>  
  
- <span data-ttu-id="ba728-106">在启用跟踪的情况下编译代码。</span><span class="sxs-lookup"><span data-stu-id="ba728-106">Compile your code with tracing enabled.</span></span> <span data-ttu-id="ba728-107">请参阅[如何：使用跟踪和调试进行条件编译](../debug-trace-profile/how-to-compile-conditionally-with-trace-and-debug.md)，了解有关启用跟踪所需编译器开关的详细信息。</span><span class="sxs-lookup"><span data-stu-id="ba728-107">See [How to: Compile Conditionally with Trace and Debug](../debug-trace-profile/how-to-compile-conditionally-with-trace-and-debug.md) for more information about the compiler switches required to enable tracing.</span></span>  
  
- <span data-ttu-id="ba728-108">为跟踪输出指定目标。</span><span class="sxs-lookup"><span data-stu-id="ba728-108">Specify a destination for tracing output.</span></span>  
  
- <span data-ttu-id="ba728-109">配置网络跟踪的行为。</span><span class="sxs-lookup"><span data-stu-id="ba728-109">Configure the behavior of network tracing.</span></span> <span data-ttu-id="ba728-110">请参阅[如何：配置网络跟踪](how-to-configure-network-tracing.md)，了解详细信息。</span><span class="sxs-lookup"><span data-stu-id="ba728-110">See [How to: Configure Network Tracing](how-to-configure-network-tracing.md) for detailed information.</span></span>  
  
 <span data-ttu-id="ba728-111">最常见的跟踪目标（也称为跟踪侦听器）是默认侦听器和日志文件。</span><span class="sxs-lookup"><span data-stu-id="ba728-111">The most common trace destinations, also referred to as trace listeners, are the default listener and the log file.</span></span>  
  
 <span data-ttu-id="ba728-112">如果不指定跟踪侦听器，则跟踪使用默认侦听器。</span><span class="sxs-lookup"><span data-stu-id="ba728-112">Tracing uses the default listener if you do not specify a trace listener.</span></span> <span data-ttu-id="ba728-113">可通过在启用托管代码的调试器中执行代码，查看发送到默认侦听器的消息，例如 .NET Framework SDK 附带的 CLR 调试器 或 Windows SDK 附带的 DBwin32.exe。</span><span class="sxs-lookup"><span data-stu-id="ba728-113">You can view messages sent to the default listener by executing your code in a managed code-enabled debugger such as the CLR Debugger shipped with the .NET Framework SDK, or DBwin32.exe shipped with the Windows SDK.</span></span> <span data-ttu-id="ba728-114">使用 CLR 调试器时，跟踪消息将出现在“输出”窗口中。</span><span class="sxs-lookup"><span data-stu-id="ba728-114">Using the CLR Debugger, the trace messages appear in the **Output** window.</span></span>  
  
 <span data-ttu-id="ba728-115">如果希望使用文件接收跟踪，则可使用配置设置指定日志文件，如以下示例所示。</span><span class="sxs-lookup"><span data-stu-id="ba728-115">If you prefer to use a file to receive traces, you can specify a log file by using configuration settings, as shown in the following example.</span></span> <span data-ttu-id="ba728-116">（有关配置文件的一般讨论，请参阅[配置文件](../configure-apps/index.md)。）</span><span class="sxs-lookup"><span data-stu-id="ba728-116">(For a general discussion of configuration files, see [Configuration Files](../configure-apps/index.md).)</span></span>  
  
 <span data-ttu-id="ba728-117">若要将跟踪发送到日志文件，请将以下节点添加到相应配置文件（应用程序或计算机）的 `<system.diagnostics>` 节点。</span><span class="sxs-lookup"><span data-stu-id="ba728-117">To send traces to a log file, add the following node to the `<system.diagnostics>` node of the appropriate configuration file (application or machine).</span></span> <span data-ttu-id="ba728-118">可按需更改文件的名称 (trace.log)。</span><span class="sxs-lookup"><span data-stu-id="ba728-118">You can change the name of the file (trace.log) to suit your needs.</span></span>  
  
```xml  
<system.diagnostics>  
  <trace autoflush="true" indentsize="4">  
    <listeners>  
      <add name="file" type="System.Diagnostics.TextWriterTraceListener" initializeData="trace.log"/>  
    </listeners>
  </trace>  
</system.diagnostics>  
```

# <a name="compiler-warning-level-1-cs1635"></a><span data-ttu-id="c9e6d-103">编译器警告（等级 1）CS1635</span><span class="sxs-lookup"><span data-stu-id="c9e6d-103">Compiler Warning (level 1) CS1635</span></span>
<span data-ttu-id="c9e6d-104">“警告代码”警告已全局禁用，无法还原</span><span class="sxs-lookup"><span data-stu-id="c9e6d-104">Cannot restore warning 'warning code' because it was disabled globally</span></span>  
  
 <span data-ttu-id="c9e6d-105">如果你使用 **/nowarn** 命令行选项或项目设置在整个编译单元中禁用某一警告，而又使用 `#pragma warning restore` 尝试还原该警告，则会出现此警告。</span><span class="sxs-lookup"><span data-stu-id="c9e6d-105">This warning occurs if you use the **/nowarn** command line option or project setting to disable a warning for the entire compilation unit, but you use `#pragma warning restore` to attempt to restore that warning.</span></span> <span data-ttu-id="c9e6d-106">若要解决此错误，对于通过命令行或项目设置禁用的所有警告，请删除其 /nowarn 命令行选项或项目设置，或删除 `#pragma warning restore` 。</span><span class="sxs-lookup"><span data-stu-id="c9e6d-106">To resolve this error, remove the /nowarn command line option or project setting, or remove the `#pragma warning restore` for any warnings you are disabling via the command line or project settings.</span></span> <span data-ttu-id="c9e6d-107">有关详细信息，请参阅 [#pragma warning](../language-reference/preprocessor-directives/preprocessor-pragma-warning.md) 主题。</span><span class="sxs-lookup"><span data-stu-id="c9e6d-107">For more information, see the [#pragma warning](../language-reference/preprocessor-directives/preprocessor-pragma-warning.md) topic.</span></span>  
  
 <span data-ttu-id="c9e6d-108">下面的示例生成 CS1635：</span><span class="sxs-lookup"><span data-stu-id="c9e6d-108">The following sample generates CS1635:</span></span>  
  
```csharp  
// CS1635.cs  
// compile with: /w:1 /nowarn:162  
  
enum MyEnum {one=1,two=2,three=3};  
  
class MyClass  
{  
    public static void Main()  
    {  
#pragma warning disable 162  
  
    if (MyEnum.three == MyEnum.two)  
        System.Console.WriteLine("Duplicate");  
  
#pragma warning restore 162  
    }  
}  
```

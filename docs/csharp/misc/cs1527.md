# <a name="compiler-error-cs1527"></a><span data-ttu-id="4251e-103">编译器错误 CS1527</span><span class="sxs-lookup"><span data-stu-id="4251e-103">Compiler Error CS1527</span></span>

<span data-ttu-id="4251e-104">命名空间中定义的元素无法显式声明为 private、protected、protected internal 或 private protected。</span><span class="sxs-lookup"><span data-stu-id="4251e-104">Elements defined in a namespace cannot be explicitly declared as private, protected, protected internal or private protected.</span></span>
  
 <span data-ttu-id="4251e-105">命名空间中的类型声明可以具有 [公共](../language-reference/keywords/public.md) 或 [内部](../language-reference/keywords/internal.md) 访问。</span><span class="sxs-lookup"><span data-stu-id="4251e-105">Type declarations in a namespace can have either [public](../language-reference/keywords/public.md) or [internal](../language-reference/keywords/internal.md) access.</span></span> <span data-ttu-id="4251e-106">如果未指定任何可访问性，则默认设置为 **内部** 。</span><span class="sxs-lookup"><span data-stu-id="4251e-106">If no accessibility is specified, **internal** is the default.</span></span>  
  
 <span data-ttu-id="4251e-107">以下示例生成 CS1527：</span><span class="sxs-lookup"><span data-stu-id="4251e-107">The following sample generates CS1527:</span></span>  
  
```csharp  
// CS1527.cs  
namespace Sample  
{  
   private class C1 {}             // CS1527  
   protected class C2 {}           // CS1527  
   protected internal class C3 {}  // CS1527  
   private protected class C4 {}   // CS1527
}  
```  
  
 <span data-ttu-id="4251e-108">以下示例将生成 CS1527，因为当程序代码中没有显式声明命名空间时，所有类型声明均隐式存在于全局命名空间内。</span><span class="sxs-lookup"><span data-stu-id="4251e-108">The following example generates CS1527 because when no namespace is explicitly declared in your program code, all type declarations are located implicitly within the global namespace.</span></span>  
  
```csharp  
//cs1527_2.cs  
using System;  
  
protected class C4 {}  
private struct S1 {}  
```

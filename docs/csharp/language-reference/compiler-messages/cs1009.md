# <a name="compiler-error-cs1009"></a><span data-ttu-id="e3ade-103">编译器错误 CS1009</span><span class="sxs-lookup"><span data-stu-id="e3ade-103">Compiler Error CS1009</span></span>

<span data-ttu-id="e3ade-104">转义序列无法识别</span><span class="sxs-lookup"><span data-stu-id="e3ade-104">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="e3ade-105">在[字符串](../builtin-types/reference-types.md#the-string-type)中，反斜杠 (\\) 的后面是一个意外字符。</span><span class="sxs-lookup"><span data-stu-id="e3ade-105">An unexpected character follows a backslash (\\) in a [string](../builtin-types/reference-types.md#the-string-type).</span></span> <span data-ttu-id="e3ade-106">编译器需要一个有效的转义字符。</span><span class="sxs-lookup"><span data-stu-id="e3ade-106">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="e3ade-107">有关详细信息，请参阅[字符转义](../../../standard/base-types/character-escapes-in-regular-expressions.md)。</span><span class="sxs-lookup"><span data-stu-id="e3ade-107">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="e3ade-108">下面的示例生成 CS1009。</span><span class="sxs-lookup"><span data-stu-id="e3ade-108">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="e3ade-109">发生该错误的原因通常是在文件名中使用了反斜杠字符，如下面的示例所示。</span><span class="sxs-lookup"><span data-stu-id="e3ade-109">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="e3ade-110">若要纠正该错误，请使用“\\\\”或 @-quoted 字符串文本，如下面的示例所示。</span><span class="sxs-lookup"><span data-stu-id="e3ade-110">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```

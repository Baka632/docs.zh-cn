 <span data-ttu-id="7bd0f-106">如果未显式初始化结构成员，则将它们初始化为其默认值。</span><span class="sxs-lookup"><span data-stu-id="7bd0f-106">Struct members are initialized to their default value if you do not explicitly initialize them.</span></span> <span data-ttu-id="7bd0f-107">类类型（和其他引用类型）的默认值为 null。</span><span class="sxs-lookup"><span data-stu-id="7bd0f-107">The default value for class types (and other reference types) is null.</span></span> <span data-ttu-id="7bd0f-108">如果尝试访问类之前，还未初始化类，则将在运行时引发 `NullReferenceException`。</span><span class="sxs-lookup"><span data-stu-id="7bd0f-108">If the class is not initialized before any attempt to access it, a `NullReferenceException` will be thrown at runtime.</span></span> <span data-ttu-id="7bd0f-109">编译器无法明确地确定是否会初始化类成员，因此 CS1060 是警告而不是错误。</span><span class="sxs-lookup"><span data-stu-id="7bd0f-109">The compiler cannot determine definitively whether the class member will be initialized or not, and so CS1060 is a warning and not an error.</span></span>

## <a name="example"></a><span data-ttu-id="7bd0f-112">示例</span><span class="sxs-lookup"><span data-stu-id="7bd0f-112">Example</span></span>  

 <span data-ttu-id="7bd0f-113">下面的代码生成 CS1060，因为类的类型 `U` 是 `struct S` 的成员，但从未进行过初始化。</span><span class="sxs-lookup"><span data-stu-id="7bd0f-113">The following code generates CS1060 because the class type `U` is a member of the `struct S` but is never initialized.</span></span>  
  
```csharp  
// cs1060.cs  
namespace CS1060  
{
    public class U  
    {  
        public int i;  
    }  
  
    public struct S  
    {  
        public U u;  
        // Add constructor to correct the error.  
        //public S(int val)  
        //{  
        //    u = new U() { i = val };  
        //}  
    }  
    public class Test  
    {  
        static void Main()  
        {  
            S s;  
            s.u.i = 5;  // CS1060  
  
            //Try these lines instead, and uncomment the constructor in S  
            // S s = new S(0);  
            // s.u.i = 5;  
  
        }  
    }
}  
```

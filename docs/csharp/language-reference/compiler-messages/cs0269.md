# <a name="compiler-error-cs0269"></a><span data-ttu-id="b4bf2-103">编译器错误 CS0269</span><span class="sxs-lookup"><span data-stu-id="b4bf2-103">Compiler Error CS0269</span></span>

<span data-ttu-id="b4bf2-104">使用了未赋值的 out 参数“parameter”</span><span class="sxs-lookup"><span data-stu-id="b4bf2-104">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="b4bf2-105">编译器无法验证在使用 out 参数之前是否已给它赋值，它的值在赋值时可能未定义。</span><span class="sxs-lookup"><span data-stu-id="b4bf2-105">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="b4bf2-106">在访问值之前，请确保为调用的方法中的 `out` 参数赋值。</span><span class="sxs-lookup"><span data-stu-id="b4bf2-106">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="b4bf2-107">如果需要使用传入的变量的值，请改用 `ref` 参数。</span><span class="sxs-lookup"><span data-stu-id="b4bf2-107">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="b4bf2-108">有关详细信息，请参阅[传递参数](../../programming-guide/classes-and-structs/passing-parameters.md)。</span><span class="sxs-lookup"><span data-stu-id="b4bf2-108">For more information, see [Passing Parameters](../../programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="b4bf2-109">示例</span><span class="sxs-lookup"><span data-stu-id="b4bf2-109">Example</span></span>  

 <span data-ttu-id="b4bf2-110">下面的示例生成 CS0269：</span><span class="sxs-lookup"><span data-stu-id="b4bf2-110">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="b4bf2-111">示例</span><span class="sxs-lookup"><span data-stu-id="b4bf2-111">Example</span></span>  

 <span data-ttu-id="b4bf2-112">如果变量的初始化发生在 try 块中，则也会出现此错误，因为编译器无法验证 try 块是否可以成功执行：</span><span class="sxs-lookup"><span data-stu-id="b4bf2-112">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```

# <a name="compiler-error-cs1943"></a><span data-ttu-id="12b45-103">编译器错误 CS1943</span><span class="sxs-lookup"><span data-stu-id="12b45-103">Compiler Error CS1943</span></span>
<span data-ttu-id="12b45-104">“type”类型的表达式不允许存在于“type”源类型的查询表达式中的后续 from 子句中。</span><span class="sxs-lookup"><span data-stu-id="12b45-104">An expression of type 'type' is not allowed in a subsequent from clause in a query expression with source type 'type'.</span></span> <span data-ttu-id="12b45-105">对“method”的调用中，类型推理失败。</span><span class="sxs-lookup"><span data-stu-id="12b45-105">Type inference failed in the call to 'method'.</span></span>  
  
 <span data-ttu-id="12b45-106">所有范围变量都必须表示可查询类型。</span><span class="sxs-lookup"><span data-stu-id="12b45-106">All range variables must represent queryable types.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="12b45-107">更正此错误</span><span class="sxs-lookup"><span data-stu-id="12b45-107">To correct this error</span></span>  
  
1. <span data-ttu-id="12b45-108">请确保该类型是实现 `IEnumerable`、`IEnumerable<T>` 或派生接口的可查询类型，或具有为之定义的查询模式的任何其他类型。</span><span class="sxs-lookup"><span data-stu-id="12b45-108">Make sure that the type is a queryable type that implements `IEnumerable`, `IEnumerable<T>` or a derived interface, or any other type which has a query pattern defined for it.</span></span>

## <a name="example"></a><span data-ttu-id="12b45-110">示例</span><span class="sxs-lookup"><span data-stu-id="12b45-110">Example</span></span>  
 <span data-ttu-id="12b45-111">下面的代码生成 CS1943：</span><span class="sxs-lookup"><span data-stu-id="12b45-111">The following code generates CS1943:</span></span>  
  
```csharp  
// cs1943.cs  
using System.Linq;  
class Test  
{  
    class TestClass  
    { }  
    static void Main()  
    {  
        int[] nums = { 0, 1, 2, 3, 4, 5 };  
        TestClass tc = new TestClass();  
  
        var x = from n in nums  
                from s in tc // CS1943  
                select n + s;  
    }  
}  
```

<span data-ttu-id="f0d11-105">希望不同类的对象使用相同代码时，类之间的转换非常有用。</span><span class="sxs-lookup"><span data-stu-id="f0d11-105">Conversions between classes are useful when you want objects of different classes to work with the same code.</span></span> <span data-ttu-id="f0d11-106">但是，协同工作的两个类不能有相互转换和多余转换，也不能有隐式转换。</span><span class="sxs-lookup"><span data-stu-id="f0d11-106">However, two classes that work together cannot have mutual and redundant conversions, or no implicit conversions.</span></span> <span data-ttu-id="f0d11-107">`class1` 和 `class2` 的类型是独立确定的，选择更全面的类型作为条件表达式的类型。</span><span class="sxs-lookup"><span data-stu-id="f0d11-107">The types of `class1` and `class2` are determined independently, and the more general type is selected as the type of the conditional expression.</span></span> <span data-ttu-id="f0d11-108">有关类型确定方式的详细信息，请参阅[条件运算符不能隐式转换](https://stackoverflow.com/questions/2215745/conditional-operator-cannot-cast-implicitly/2215959#2215959)。</span><span class="sxs-lookup"><span data-stu-id="f0d11-108">For more information about how types are determined, see [Conditional operator cannot cast implicitly](https://stackoverflow.com/questions/2215745/conditional-operator-cannot-cast-implicitly/2215959#2215959).</span></span>

<span data-ttu-id="f0d11-109">若要解决 CS0173，请确保 `class1` 和 `class2` 之间有且仅有一个隐式转换，而不论向哪个方向进行转换或在哪个类中进行转换。</span><span class="sxs-lookup"><span data-stu-id="f0d11-109">To resolve CS0173, verify that there is one and only one implicit conversion between `class1` and `class2`, regardless of which direction the conversion is in and regardless of which class the conversion is in.</span></span> <span data-ttu-id="f0d11-110">有关详细信息，请参阅[用户定义转换运算符](../operators/user-defined-conversion-operators.md)和[内置数值转换](../builtin-types/numeric-conversions.md)。</span><span class="sxs-lookup"><span data-stu-id="f0d11-110">For more information, see [User-defined conversion operators](../operators/user-defined-conversion-operators.md) and [Built-in numeric conversions](../builtin-types/numeric-conversions.md).</span></span>

## <a name="example"></a><span data-ttu-id="f0d11-111">示例</span><span class="sxs-lookup"><span data-stu-id="f0d11-111">Example</span></span>

<span data-ttu-id="f0d11-112">下面的示例生成编译器错误 CS0173：</span><span class="sxs-lookup"><span data-stu-id="f0d11-112">The following examples generate compiler error CS0173:</span></span>

```csharp
public class C {}

public class A
{
    // The following code defines an implicit conversion operator from
    // type C to type A.
    //public static implicit operator A(C c)
    //{
    //    A a = new A();
    //    a = c;
    //    return a;
    //}
}

public class MyClass
{
    public static void F(bool b)
    {
        A a = new A();
        C c = new C();

        // The following line causes CS0173 because there is no implicit
        // conversion from a to c or from c to a.
        object o = b ? a : c;

        // To resolve the error, you can cast a and c.
        // object o = b ? (object)a : (object)c;

        // Alternatively, you can add a conversion operator from class C to
        // class A, or from class A to class C, but not both.
    }

   public static void Main()
   {
      F(true);
   }
}
```

```csharp
class M
{
    static int Main ()
    {
        int X = 1;
        // The following line causes CS0173.
        object o = (X == 0) ? null : null;
        return -1;
    }
}
```
